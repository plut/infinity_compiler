    pub level: usize,
    pub name: String,
    pub table_type: TableType,

Expand description

The full database description of a game resource.

This contains all relevant information to fully define a resource on the SQL side.

In practice there exists exactly one Schema instance per resource, and it is compiled by the SqlRow derive macro.

Fields§

§level: usize

The nesting level for this resource (0 for top-level resources).

§name: String

Name of the SQL table mapped to this data.

§table_type: TableType

Position of the schema in the arborescence.

§fields: Fields

Description of the SQL table for this data.

pub fn create_tables_and_views<T>(
    &self,
    f: impl Fn(String) -> Result<T>
) -> Result<()>

Creates all tables and views in the database associated with a given resource.

The tables are as follows:

  • load_{name}: data read from game files.
  • add_{name}: new data inserted by mods.
  • edit_{name}: data modified by mods. The views are:
  • {name}: the main view on which edits are done by mods (and propagated to add_{name} or edit_{name} by triggers as needed).
  • save_{name}: the view used to save new or modified values to game files.
source

pub fn insert_sql(&self, prefix: impl Display) -> String

The SQL code for inserting into a table.

This is used for both initial populating of the database from game files and for simod.insert.

The or parameter allows calling SQL INSERT OR IGNORE; this is used when initializing the database to ignore resources which are superseded by override files.
